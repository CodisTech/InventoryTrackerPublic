Security Features

CSRF Protection: All state-changing operations (POST, PATCH, DELETE) require valid CSRF tokens
Secure HTTP Headers: Content-Security-Policy, X-XSS-Protection, X-Content-Type-Options, and more
Session Security: HttpOnly, Secure flags, SameSite restrictions, and 4-hour timeout
Rate Limiting: API endpoints protected from brute force and DDoS attacks
Role-Based Access Control: Permissions strictly enforced based on user roles
Note: This GitHub Pages deployment is for demonstration purposes only. The full application requires a backend server to function properly.

Getting Started

To run the full application locally:

  1. Clone the repository: git clone https://github.com/codistech/inventory-management-system.git
  2. Install dependencies: npm install
  3. Start the development server: npm run dev
  4. Access the application at http://localhost:5000
